회원 컬러칩 고도화 — 30+ 회원 + 다코치 시스템 대비

현재 10색 팔레트는 회원이 11명을 넘는 순간 색 충돌이 시작됩니다. 또한 시스템이 PT를 넘어 필라테스·요가·골프·테니스 등 여러 코칭 도메인으로 확장된다면, 색 자체의 의미와 톤도 도메인별로 달라져야 합니다. 4가지 컨셉을 비교하고, 추천안 + 도메인별 프리셋 + 토큰 명세까지 정리했습니다.

0해결하려는 문제

10색 팔레트의 한계와, 다코치 확장 시 추가로 생기는 요구사항.

현재의 마찰점

  • 11번째 회원부터 색이 무조건 중복 — 트레이너 1명이 보통 20-40명을 관리하므로 같은 색 회원이 2-4명씩 겹칩니다.
  • 해시 배정의 클러스터링 — 신규 5명을 등록하면 같은 색이 2-3명에 몰리는 일이 잦습니다.
  • 색만으로는 식별 불가 — 캘린더 좁은 칸에 "민수/민지/민영"이 동시에 있으면 색 톤만으로 누가 누군지 모릅니다.
  • 도메인 톤이 단일 — 골프 코치가 PT용 채도 높은 팔레트를 쓰면 자연스럽지 않을 수 있습니다.

1컨셉 A. 휴(Hue) → 명도(Shade) 2단 선택 추천

사용자가 제안한 방향. 색 클릭 → 명도 트레이가 펼쳐지고 → 명도 선택. 10 휴 × 4 명도 = 40색까지 충돌 없이 확장됩니다.

인터랙션 — 휴를 클릭하면 명도가 펼쳐집니다

실제로 클릭해 보세요. 회색 점선 마커가 있는 칩은 이미 사용 중인 색입니다 (자동 회피).

CLICK A HUE → EXPANDS
선택된 색 없음
  • 1단계 (휴): 기존 10색 그대로 — 학습 비용 0. 우상단에 "사용 중 N명" 작은 카운트 배지.
  • 2단계 (명도): 4단계 (어두움 / 기본 / 밝음 / 더 밝음). 이미 쓰인 명도는 해칭 + ●로 표시.
  • 애니메이션: 휴는 1.18배 확대 + 외곽 링 / 명도 트레이는 좌→우 0.28s 펼침.
  • 키보드: 휴 포커스 후 Enter 펼침, ←→ 명도 이동, Esc 닫기.
왜 추천? ① 학습 비용이 가장 낮음(기존 멘탈 모델 유지) ② 색 정체성 유지(같은 휴=같은 회원군 느낌) ③ 명도는 다크모드/캘린더 배경과의 대비도 같이 조절할 수 있는 부가 효과.

스마트 자동 배정 — 라운드로빈 + 휴 우선 + 명도 후순위

신규 회원에게 색을 자동 배정할 때, 가장 적게 쓰인 휴 → 그 안에서 가장 적게 쓰인 명도 순으로 결정. 결정적이라 마이그레이션 안전.

의사 코드 보기 
function autoAssign(members, palette) {
  // palette: [{hue:'sky', shades:['#0c4a6e','#0284c7','#38bdf8','#bae6fd']}, ...]
  const used = countByHueShade(members);

  // 1. 가장 적게 쓰인 휴 후보들
  const minHueCount = Math.min(...palette.map(h => used.hue[h.id] ?? 0));
  const hueCandidates = palette.filter(h => (used.hue[h.id] ?? 0) === minHueCount);

  // 2. 이름 해시로 결정적 휴 선택 (tie-break)
  const hue = hueCandidates[hash(name) % hueCandidates.length];

  // 3. 그 휴 안에서 가장 적게 쓰인 명도
  const shadeCounts = hue.shades.map(s => used.shade[s] ?? 0);
  const minShade = Math.min(...shadeCounts);
  const shadeIdx = shadeCounts.findIndex(c => c === minShade);

  return hue.shades[shadeIdx];
}

3컨셉 C. 색 + 이니셜 모노그램 병행 권장

컨셉 A와 같이 쓰는 것을 추천. 색은 그룹핑 단서, 이니셜은 절대 식별자. 10명 이상에서도 누가 누군지 즉시 보입니다.

5다코치 시스템 — 도메인별 팔레트 프리셋

시스템이 PT뿐 아니라 다양한 코칭으로 확장되는 것을 가정. 도메인별로 분위기는 달라도 구조(10휴 × 4명도)는 동일해야 컴포넌트 재사용성이 유지됩니다.

5-1. 도메인 프리셋 5종 제안

각 프리셋은 동일한 토큰 구조(--hue-1 ~ --hue-10, 각 4명도)를 따르며, 워크스페이스 설정에서 한 번에 전환됩니다.

5-2. 코치 도메인이 시스템 전반에 미치는 다른 영향

팔레트만 바꾸는 게 아니라, 도메인별로 자연스러운 용어·아이콘·기본 세션 길이도 달라집니다. 프리셋에 함께 묶어 두면 "코치 가입 → 도메인 선택 → 즉시 사용"이 됩니다.
