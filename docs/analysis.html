레슨핏 디자인 시스템 진단 & 고도화 제안

1토큰 · 컬러 시스템

현재는 단일 라이트 테마 + 정적 토큰만 정의되어 있습니다. 스케일·테마·시맨틱을 한 번 정리할 시점입니다.

1-1. --success·--success-bg 토큰화 Quick win

완료 상태 색 #16a34a가 모달 버튼에 하드코딩되어 있고, --warn은 정의만 되어 사용처가 없습니다. 시맨틱 4종(info/success/warn/danger)을 대칭으로 갖추면 향후 토스트·노쇼·만료 알림이 일관됩니다.

토큰제안 값활용
--success#16a34a완료 상태, 출석률 양호
--success-bg#dcfce7완료 배지·토스트 배경
--warn#f59e0b노쇼, 만료 임박, 결제 미수
--warn-bg#fef3c7경고 배지(현재 "오늘 헤더"에 비공식 사용 중)
주의: #fef3c7는 이미 "오늘 헤더 셀"에 쓰이고 있는데 토큰화되지 않아 의미가 모호합니다. --today-bg로 별도 명명하거나 --warn-bg로 통합 — 둘 중 하나로 결정해 두세요.

1-2. 다크 모드를 위한 시맨틱 분리 High

야간에 회원 응대를 자주 하는 트레이너 특성상 다크 모드 수요가 큽니다. hex가 토큰에 직접 박혀 있으면 테마 분기가 어려워집니다. 표면(surface)·전경(text)·강조(accent) 3계층으로 정리해 두 테마 모두 같은 컴포넌트가 동작하게 만드는 것을 권장합니다.

1-3. 회원 팔레트 — 색약 사용자 충돌 점검 Medium

흰 텍스트 대비는 모두 통과했지만, 색끼리 구분은 별개 문제입니다. 적녹 색약(deuteranopia) 시뮬을 걸면 산호↔귤, 숲↔머스터드 구분이 약해집니다.

1-4. 색 자동 배정 해시 → 라운드로빈 Medium

현재 colorFor(name) = hash % 10은 결정적이지만, 신규 회원 3-4명이 같은 색에 몰리는 일이 흔합니다. "가장 적게 쓰인 색" 우선 배정으로 충돌을 최소화할 수 있습니다.

구현 힌트: 회원 등록 시 현재 색 카운트 → 최저값 중 하나를 결정적으로 선택(이름 해시 tie-break). 결정적이라 마이그레이션 안전.

1-5. 스페이싱·반경 스케일 토큰화 High

2타이포 · 가독성

14px 본문은 데스크톱 표 안에서는 적절하지만, 회원 상세·세션 메모처럼 읽기 중심 화면에서는 작습니다.

2-1. 본문 기본 15px / 1.6로 Quick win

40대 이상 트레이너 비율을 고려하면 14px는 부담입니다. 모달 등 좁은 영역만 14px로 두고, 일반 본문은 15px가 적절합니다. line-height를 1.6~1.65로 늘리면 한글 가독성이 눈에 띄게 좋아집니다.

2-2. 타입 스케일 + tabular-nums High

지금은 "이벤트 이름 12px / 시간 10px / 모달 제목 18px"가 컴포넌트별로 흩어져 있습니다. 6단계 스케일로 토큰화하고, 시간·횟수·금액에 font-variant-numeric: tabular-nums를 적용하면 캘린더와 통계 화면에서 숫자가 흔들리지 않습니다.

2-3. Noto Sans KR 가변 폰트 + 사전 로드 Medium

현재 문서엔 폰트 로딩 전략이 없습니다. PWA 첫 로드를 빠르게 하려면 (1) font-display: swap, (2) Noto Sans KR Variable로 weight 5종을 한 파일에, (3) <link rel="preload">로 본문 weight만 우선 로드 — 이 3가지를 명세에 박아 두세요.

3접근성

색 대비는 잘 잡혀 있지만, 키보드·스크린리더·모션 가이드가 빠져 있습니다.

3-1. focus-visible 토큰 통일 Critical

현재 포커스 링이 "테이블 셀 인풋"에만 명시되어 있고 버튼·링크·스와치엔 없습니다. 키보드로 앱을 돌리면 어디 있는지 알 수 없는 상태가 됩니다.

FOCUS RING · PROPOSED
outline 2px + offset 2px + soft glow
토큰 제안: --focus-ring: 0 0 0 4px rgba(37,99,235,.18); --focus-outline: 2px solid var(--primary); + :focus-visible로만 적용 (마우스 클릭 시엔 표시 안 함).

3-2. 상태의 이중 부호화 강화 High

"완료=초록, 취소=빨강, 예정=어두운 회색"은 색에만 의존합니다. 완료에 ✓ 배지, 취소에 해칭 패턴은 좋은 시작이고, 예정/노쇼/지난까지 포함해 모두 아이콘이나 패턴이 같이 있어야 색약·흑백 인쇄에서도 통합니다.

상태색패턴/아이콘스크린리더

3-3. prefers-reduced-motion·prefers-contrast 지원 Medium

모달 페이드, 토스트 슬라이드, 캘린더 이동 애니메이션을 reduced-motion 환경에서 0.01ms로 단축하는 가드를 추가하세요. 토큰 파일 끝에 @media (prefers-reduced-motion: reduce) 한 블록이면 끝입니다.

3-4. PIN 잠금 — 스크린리더·키패드 안내 Low

aria-label="PIN 입력"·inputmode="numeric"·autocomplete="off"를 함께 명세에 박아 두세요. 모바일 숫자 키패드 자동 전환·1Password 자동완성 차단을 한 번에 해결합니다.

그룹 PT나 동시간대 다회원 운영 시 layoutEventsInColumn의 lane이 5+가 되면 한 칸이 너무 좁아집니다. 4개를 넘는 순간 "+3 더보기"로 접고, 클릭 시 팝오버에 전체 리스트를 보여주는 패턴이 안정적입니다.

5-5. 휴식·이동·식사 블록 패턴 Low

현재 "개인 일정"은 회색 + 대각선 줄무늬 한 종류뿐인데, 트레이너 일과는 점심·이동·휴식·관리 업무 등으로 세분됩니다. 회색 톤 안에서 밀도(텍스처)로 4종 정도 구분 가능한 보조 패턴 셋을 미리 정의해 두면 자기 시간 보호에 도움이 됩니다.

6모바일 · 반응형

트레이너는 현장에서 폰으로 가장 많이 씁니다. 마찰 줄일 지점이 몇 군데 보입니다.

6-1. 컬러 팔레트 칩 터치 타깃 High

≤480px에서 10열 그리드로 유동폭 전환은 좋은데, 28px 칩은 손가락에 작습니다. 모바일에서는 36px+ 또는 5열 × 2행로 재배치하는 것을 권장합니다.

PALETTE · MOBILE 5×2

6-2. 스와이프 액션 — 회원 행 / 세션 블록 Medium

회원 리스트 행을 좌측으로 슬라이드 → "세션 추가 · 메모 · 삭제" 액션 노출. 아젠다 뷰의 세션을 슬라이드 → "완료 · 노쇼 · 취소" 토글. 모바일 네이티브 PT 앱 패턴이라 학습 곡선이 사실상 0입니다.

6-3. safe-area-inset 명세 Low

PWA로 홈화면 추가 시 iPhone 노치·하단 인디케이터 영역에 콘텐츠가 가려질 수 있습니다. env(safe-area-inset-*) 사용을 토큰에 반영하세요. 특히 하단 고정 CTA 사용 시 필수.

7데이터 · 인사이트 (고도화)

단순 캘린더에서 "운영 도구"로 한 단계 올라가려면 회원과 시간을 보는 새 화면이 필요합니다.

7-1. 회원별 출석 히트맵 High

각 회원의 카드 상단에 최근 12주 × 7요일 격자로 출석 패턴을 보여줍니다. 색 강도 = 그 주 세션 수. 어느 회원이 빠지기 시작했는지 한눈에 잡힙니다.

7-3. 회원 리텐션 시그널 Medium

"3주 연속 결석 ↓ 추세", "재등록 가능성 높음(완료 8/10)" 같은 룰 기반 라벨을 회원 카드 옆에 자동 부여. 트레이너가 먼저 연락할 회원을 매주 한눈에 골라낼 수 있게 해줍니다. 과한 ML보다는 5-6개 룰셋으로 시작.

8오프라인 · 동기화

현장 와이파이는 자주 흔들립니다. SW 캐시 전략은 잘 잡혔지만, 사용자에게 보이는 UI가 빠졌습니다.

8-1. 오프라인 인디케이터 High

네트워크 끊김을 헤더에 1줄 띠로 표시. "오프라인 · 변경은 자동 저장됩니다 · 동기화 대기 3건". 복귀 시 슬라이드 업으로 사라지고 토스트로 "동기화 완료".

8-2. 동기화 충돌 해소 UI Medium

같은 세션을 두 기기(PC 사무실 / 폰 현장)에서 동시에 수정하면 충돌이 발생할 수 있습니다. "서버 vs 내 변경"을 좌우로 보여주고 한쪽 선택 또는 병합. Supabase row 단위로 대부분 해결되지만, UI가 없으면 트레이너는 그냥 데이터가 사라졌다고 느낍니다.

8-3. SW 업데이트 알림 Low

새 캐시 버전(pt-cache-vN+1) 발견 시 우하단에 "새 버전이 준비되었습니다 [새로고침]" 토스트. 강제 리로드는 절대 금물 — 트레이너가 모달 입력 중일 수 있음.

매일 같은 작업을 반복하는 도구는 단축키 한두 개만 있어도 만족도가 폭발합니다.

9-1. 커맨드 팔레트 ⌘K Medium

9-2. 반복 일정 템플릿 High

"매주 화·목 19시, 8주, 김민수" 같은 반복 룰을 한 번에 생성. 현재 1건씩 입력 가정이라면 신규 회원 1명당 16번 클릭이 필요한데, 템플릿 1번이면 끝.

10우선순위 로드맵

개선 항목을 ROI 기준으로 3단계로 묶었습니다. 각 단계는 1-2주 작업 분량을 가정합니다.

NOW · 즉시 (1주)
  • focus-visible 토큰 통일
  • "노쇼" 세션 상태 추가
  • --success·--warn 토큰화
  • 본문 15px / 1.6 적용
  • 버튼 sm/md/lg 스케일
NEXT · 다음 (3-4주)
  • 다크 모드 — surface/text/accent 분리
  • 토스트 + 빈 상태 + 스켈레톤
  • 아젠다 뷰 (모바일 기본)
  • 드래그로 일정 이동
  • 오프라인 인디케이터 + 회원 출석 히트맵
  • 스페이싱·타입 스케일 토큰화
LATER · 분기 내
  • 주간 운영 대시보드 (KPI 4)
  • 커맨드 팔레트 ⌘K + 단축키
  • 반복 일정 템플릿
  • 리텐션 시그널 룰 엔진
  • 동기화 충돌 해소 UI
  • 색약 시뮬·이중 부호화

마무리

현재 v1은 토큰 정의·의미 충돌 방지·접근성 의식 면에서 이미 모범적입니다. 이 분석의 NOW 5건만 적용해도 사용 품질이 한 단계 올라가고, NEXT까지 가면 "PT 트레이너 전용 도구"로서의 차별점이 분명해집니다. LATER 항목들은 사용자 베이스가 200명을 넘는 시점에 다시 검토해도 늦지 않습니다.
